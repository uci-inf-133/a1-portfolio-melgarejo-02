3. Did you ignore any of the warnings or errors presented by the accessibility checker? If so, why does this not seem like an accessibility concern? If it's useful, you can consolidate your thoughts on multiple warnings/errors if the rationale is similar.
Yes, I had no actual problems but I did have "Potential Problems." I reviewed them, but I felt my site had already dealt with them:
- Alt text warnings: My images use descriptive alt text that matches the content of the images. The checker repeatedly flagged the same image for multiple "may require" checks, but the description is accurate for screen readers.
- Navigation/link text warnings: Navigation links are clear and direct and links like "LinkedIn" and "Resume (PDF)" are also meaningful in context.
- Headings warnings: Headings are used to structure content rather than for visual formatting only.
- Direction/language/sensory checks: The site is written in English (lang="en"), and there are no right-to-left language sections.
- Skip link/sitemap warnings: The site has a consistent navigation bar on every page, which provides a clear way to move through the site.
